UI/UX
   UI ?
     사용자와 컴퓨터가 서로 정보를 주고받기 위해
     상호작용하는 프로그램의 화면을 User Interface
     CLI(Commend Line Interface)
     TUI(Terminal User Interface)
     GUI(Graphic User Interface)
     전자기기, 소프트웨어 화면 <-> 사용자 접하는 부분
   UX ?
     사용자의 기존 경험을 바탕으로 쉽게 접근할 수 있는
     UI 를 구성하는 디자인 기법
     반응형 웹 (컴퓨터 화면, 태블릿, 모바일)

UI 요구사항
 - 어떠한 문제의 해결 또는 목적 달성을 위하여
   고객에 의해서 요구되는 사항
 - 프로그램이 수행해야할 기능들이 가져야할 서비스,
   제약사항 들을 명시해 놓는 것
 - 고객이 요구하거나 요구하지 않았더라도
   당연히 제공되어야 하는 기능들이 반드시 
   포함되어 있어야 한다.

UI 요구사항 명세서 작성 필수 요소
 - 제목, 버전, 프로젝트의 명칭
 - 명세서의 내용은 고객과 개발자 모두 알기 쉽게 간결한 
    내용으로 작성해야 한다.
 - 프로젝트가 수행할 모든 기능과 시스템에서 갖추어야 할 
    서비스, 자원등이 명시되어 있어야 한다.
 - 기술되어 있는 요구사항 또는 기술들은 검증이 가능해야 하고 
   해당 프로젝트의 품질, 중요도를 측정할 수 있어야 한다.
 - 고객과 개발자 모두가 커뮤니케이션 할 수 있는 문서로
   작성해야 한다.


- UI 설계시 고려해야할 사항 - 
  - UI 설계 작업은 실제 화면과 동일하게 작업해야 한다.
  - 컨텐츠를 구성하거나 배치할 때 반드시 그 목적이 있어야 한다.
  - 사업의 전략, 컨셉, 서비스의 내용과 메인-서브 페이지의
    컨셉 등을 먼저 잡아놓고 UI 설계를 해야 한다.
  - 모든 UI 기획은 사용자의 입장에서 행해져야 한다.
  - UI 화면 설계와 관련 문서는 직관적이며 쉽게 구성되어야 한다.


 표준화 대상
   - 동선 규칙 : 사용자가 어디에서 접속하고 어디로 흘러갈 지
   - 네이밍 룰 / 레이블 규칙 : 프로젝트에서 사용할 객체나
                    특정 요소들의 이름을 통일화 하는 것
   - 화면 분할 규칙 : 프로젝트에서 화면에 표시할 내용들을 사전에 정의하는 것

 UI 설계 시 첨부해야할 문서
  - 유스 케이스 
   (프로젝트에서 제공하는 기능들을 간략한 도표로 나타낸 것)
  - 와이어 프레임
   (UI 중심의 각 페이지들의 상세 설계도)
  - 스토리보드
   (와이어 프레임, 특정 프로세스를 바탕으로 이동할 수 있는
    경로들을 명시한 것)
  - 데이터 플로우 다이어그램(Data Flow Diagram / Work Flow)
     (데이터가 어떤 페이지에서 어떤 페이지로, 
      또는 서버 내 어떤 경로로 전달되는 지 정리한 문서)
  - 메뉴 구성도 (Information Architecture)
     (접근을 위한 메뉴가 어떻게 구성되어 있어야 하는지 정리한 문서)

 관련 용어
   - 프로토타입(Prototype) : 프로젝트의 초안
   - 목업(Mockup) : 정적인 프로젝트의 모형
